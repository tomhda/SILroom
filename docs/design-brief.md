# Design Brief

## Goal

Chatworkの左端チャットルーム一覧を、グループ・未分類・固定・1:1チャットが混ざって渋滞しないUIに置き換える。

## Core Direction

案1ベースの構成にする。

```text
[細いスペースバー] [選択中スペースのチャット一覧] [本文エリア] [必要時だけ開く概要欄]
```

左端の細いスペースバーで大分類を切り替え、右隣のリストには選択中スペース内のチャットだけを出す。
ワークスペースはユーザーの作業順に合わせられるよう、左バー上でドラッグ並び替えできる。
チャットルーム一覧は作業量に合わせて横幅を広げられ、不要な時は細いルームバーに縮められる。縮小中はマウスオーバーで通常サイズの一覧を重ねて表示する。

## Spaces

- 全体/未読
- 固定
- 未分類
- マイチャット
- DM
- Chatworkカテゴリー由来のワークスペース

スペースバーは通常幅を維持し、マウスオーバー時だけラベルを横に重ねて表示する。

## DM Handling

Chatwork上では1:1チャットでも、拡張側ではDMとして扱う。

マイチャットはDMに含めず、専用スペースとして分離する。

DMはグループ・未分類・固定の通常リストから分離し、DMスペースにまとめる。これにより左ペインの渋滞を減らす。

## Room List Behavior

選択中スペースのチャット一覧は、次の優先順位で並べる。

1. 自分宛
2. 未読新着
3. 固定
4. 最近更新
5. 通常

固定チャットは重要だが、全てを常に最上部に押し上げると新着が埋もれるため、固定は視覚的な常駐性と並び替え優先度を分けて扱う。

API補助が有効な場合は、Chatwork APIの最終更新時刻を使って新着順を安定させる。API補助がない場合は、Chatwork本体DOMから取得できる範囲で並べる。

## Badges

通知バッジは必ず分ける。

- 緑: 自分宛
- 青: 通常未読
- 灰: 低優先または既読寄りの情報

グループ見出しやスペースバーにも、合算数ではなく「自分宛」と「通常未読」を分けて出す。

API補助が有効な場合は、`mention_num` を自分宛、`unread_num - mention_num` を通常未読として扱う。

## Overview Drawer

右側の概要欄は常時表示しない。

右端に細いタブだけ置き、必要な時だけドロワーとして開く。概要欄の中身は初期案として以下を想定する。

- 概要
- タスク
- ファイル
- リンク
- メンバー

閉じた状態では本文エリアを広く使う。

## Non Goals

- Chatworkのメッセージ本文機能を作り直さない
- 送信、編集、タスク作成などの業務操作は初期版では触らない
- Chatworkの通信APIを横取りする前提にしない
- 最初からネイティブアプリ化しない
